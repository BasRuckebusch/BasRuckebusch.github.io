Rogoto

Game link

Rogoto is a roguelike with automation and tower defense elements I made together with a small team for the Group Projects course at DAE.

It was a hobby award finalist in the GDWC.

Project Duration:
11 weeks
8 hours per day
Engine: Unity

Most of my efforts went into the terrain generation, a robust combination of chunks using cellular automata and a custom system of connecting those chunks with mining areas and giving the player choices between them. The chunk inside of the chunk consists of 3 major layers, caverns, the main layer and pilars. Caverns and pillars are generated as 2 big repeating cellular automata maps allowing them to travel between chunks. The main layer is a smaller bit of cave generation with borders surrounding it so it doesn't clip into other chunks.

The placement works by having a starting chunks all the other chunks connect up to in the form of branches. The first chunk of a branch is always just one but afterwards the player gets a choice of two each time, both of which have a granteed finished path as long as the branch lenght. When the player picks a chunk it's branch becomes the main branch and the other gets deleted, then a new potential branch gets generated when the player is given a choice again.

When a branch if completed it solidifies merging all the meshes in each chunk to make it a lot more performant even when there's hundereds of them in play. The last chunk fo a branch gets counted as "exhausted" meaning it can never be used as a starting chunk, this is due to it having a permanent nest which would hidner mining. Whenever a starting chunk is can't generate any new chunks next to it it becomes echausted as well and the closest non exhausted chunk becomes the starting chunk and gets unsolidified. This system allows for a spiderweb like structure to the caves with a path to the center for the enemies to follow always being possible.
